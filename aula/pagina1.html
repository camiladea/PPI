Lista de Comandos Básicos do Git

CONFIGURAÇÃO INICIAL

  • git config global user.email "camila.a09@aluno.ifsc.edu.br": Define seu e-mail para todos os repositórios Git no seu computador.
  • git config global user.name "Camila": Define seu nome de usuário globalmente.

INICIAR E ACOMPANHAR PROJETO

  • git init: Cria um novo repositório Git local no diretório atual.
  • git add .: Adiciona todas as alterações (arquivos novos ou modificados) para a área de preparação (staging).
  • git status: Mostra o status atual dos arquivos (o que foi modificado, adicionado ou está pronto para commit).
  • git commit: Cria um commit com os arquivos na área de staging. Usar sem -m abre o editor para digitar a mensagem.
  • git commit -m "app inicial": Cria um commit com a mensagem "app inicial".

HISTÓRICO

  • git log: Mostra o histórico completo de commits com detalhes.
  • git log oneline all: Mostra o histórico em uma linha por commit, incluindo todas as branches.

CONECTAR AO GITHUB

  • git remote add origin https://github.com/camiladea/PDM: Conecta o repositório local ao repositório remoto no GitHub com o nome "origin".
  • git remote -v: Mostra os repositórios remotos configurados.

ENVIAR PARA O GITHUB

  • git push origin master: Envia os commits da branch master para o repositório remoto.
  • git push origin hello: Envia os commits da branch hello para o repositório remoto.

BRANCHES

  • git branch: Lista todas as branches existentes no repositório.
  • git checkout -b hello: Cria uma nova branch chamada hello e muda para ela.
  • git checkout master: Muda para a branch master.

FLUXO BÁSICO APÓS EDIÇÃO

  • git add .: Adiciona as mudanças após editar arquivos.
  • git commit -m "mensagem": Salva as alterações com uma mensagem.
  • git push origin nome-da-branch: Envia as alterações para o GitHub.

CLONAR

  • git clone url do repositório: clona repositório.
  • git diff: exibe a diferenca entre versoes
